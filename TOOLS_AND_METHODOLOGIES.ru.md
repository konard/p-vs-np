## Обзор

Этот документ каталогизирует теоретические инструменты, математические методологии, вычислительные техники и ресурсы, доступные для изучения и потенциального решения проблемы P vs NP. Он служит всеобъемлющим руководством для исследователей, приближающихся к этому фундаментальному вопросу в вычислительной теории сложности.

### Ключевое переосмысление

**✅ ДОКАЗАНО: P ⊆ NP** — Каждая задача, разрешимая за полиномиальное время, также проверяема за полиномиальное время.
[См. формальные доказательства](proofs/p_vs_np_decidable/README.md) в Lean 4, Rocq, Isabelle/HOL и Agda.

**❓ ВОПРОС: Верно ли NP ⊆ P?**

Поскольку P ⊆ NP доказано, **P vs NP доказуемо/недоказуемо тогда и только тогда, когда NP ⊆ P доказуемо/недоказуемо.**

Этот фреймворк означает, что все инструменты и методологии должны быть направлены на понимание, может ли каждая задача, проверяемая за полиномиальное время, также быть решена за полиномиальное время.

## 1. Теоретические основы

### 1.1 Вычислительные модели

**Машины Тьюринга**
- **Цель:** Формальная модель вычислений; стандарт для определения P и NP
- **Варианты:**
  - Детерминированная машина Тьюринга с одной лентой (DTM)
  - Многоленточная машина Тьюринга
  - Недетерминированная машина Тьюринга (NTM)
  - Машина Тьюринга с оракулом
  - Чередующаяся машина Тьюринга
- **Инструменты для анализа:**
  - Временная сложность T(n) = число шагов в худшем случае
  - Пространственная сложность S(n) = использованные ячейки в худшем случае
  - Графы конфигураций
  - Вычислительные таблицы
- **Ключевые ресурсы:**
  - Sipser "Introduction to the Theory of Computation"
  - Arora-Barak "Computational Complexity: A Modern Approach"

**Булевы схемы**
- **Цель:** Неравномерная модель вычислений; центральна для нижних оценок
- **Компоненты:**
  - Вентили: AND, OR, NOT, NAND
  - Размер схемы: количество вентилей
  - Глубина схемы: длина самого длинного пути
- **Классы схем:**
  - **P/poly:** Полиномиальный размер, неравномерные схемы
  - **NC:** Полилогарифмическая глубина, полиномиальный размер
  - **AC^0:** Постоянная глубина, неограниченный fan-in
  - **TC^0:** Постоянная глубина с пороговыми вентилями
  - **ACC^0:** Постоянная глубина с модульными вентилями
- **Ключевая связь с P vs NP:**
  - Если существует язык в NP, требующий супер-полиномиальных схем, то P ≠ NP
  - Доказательство P ⊆ NP показывает, что каждая P-задача имеет полиномиальные схемы ([см. доказательства](proofs/p_vs_np_decidable/README.md))

**Рандомизированные вычисления**
- **Модель:** Алгоритмы с доступом к случайным битам
- **Классы:**
  - **BPP:** Вероятностное полиномиальное время с ограниченной ошибкой
  - **RP:** Рандомизированное полиномиальное время (односторонняя ошибка)
  - **ZPP:** Нулевая ошибка, вероятностное полиномиальное время
- **Связь с P vs NP:**
  - Гипотеза дерандомизации: BPP = P
  - Наблюдение Kabanets: либо дерандомизация, либо P ≠ NP

- Приложения:
  - Нижние оценки для алгебраических схем

  - Семидефинитное программирование

- Алгебраическая геометрия

### 2.3 Логика и теория доказательств

**Пропозициональная логика**

- Приложения:
  - Сложность доказательств

**Математическая логика**
- Теория моделей
- Теория множеств (ZFC)
- Независимость и неразрешимость
- Приложения:
  - Потенциальная независимость P vs NP от ZFC
  - Мета-математические исследования
  - Теоремы Гёделя

**Сложность доказательств**
- Фреймворк Cook-Reckhow
- Нижние оценки для систем доказательств
- Связь с NP vs coNP
- Ключевой результат: Если NP ≠ coNP, то нет системы доказательств с полиномиальными доказательствами для всех тавтологий

## 3. Техники доказательств

### 3.1 Диагонализация

**Классическая диагонализация**
- Метод Кантора
- Теоремы иерархии времени и пространства
- Доказательство неразрешимости проблемы остановки
- Успехи:
  - P ⊊ EXP (по теореме иерархии времени)
  - NP ⊊ NEXP
  - LOGSPACE ⊊ PSPACE
- **Ограничение:** Релятивизируется (барьер BGS)

**Продвинутая диагонализация**
- Отложенная диагонализация
- Диагонализация с редукцией
- Ограничения из-за релятивизации

### 3.2 Нижние оценки для схем

**Метод ограничений**
- Лемма переключения Håstad
- PARITY не в AC^0
- Экспоненциальные нижние оценки для постоянной глубины
- Применение: Доказанные нижние оценки для ограниченных моделей

**Метод аппроксимации**
- Разборов (1985): Монотонные схемы
- Аппроксимировать сложные функции простыми
- Доказать, что требуется много вентилей
- Применение: CLIQUE требует экспоненциальных монотонных схем

**Полиномиальный метод**
- Представить булевы функции как полиномы
- Использовать степень как меру сложности
- Применение: Нижние оценки для AC^0

**Техника Уильямса (алгоритмы → нижние оценки)**
- Разработать более быстрые алгоритмы для circuit-SAT
- Использовать для доказательства нижних оценок
- Результат: NEXP ⊄ ACC^0 (Williams 2011)
- **Преимущество:** Нерелятивизирующее
- **Ключевая идея:** Если класс имеет слишком малые схемы, можно решить circuit-SAT быстрее

### 3.3 Коммуникационная сложность

**Модель:**
- Два игрока с разными входами
- Минимизировать коммуникацию для вычисления функции
- Приложения:
  - Нижние оценки для схем
  - Структуры данных
  - Потоковые алгоритмы

**Методы:**
- Ранговый метод
- Дискрепансия
- Сложность информации
- Ссылки:
  - Kushilevitz-Nisan "Communication Complexity"

## 6. Специфические исследовательские программы

### 6.1 Геометрическая теория сложности (GCT)

**Цель:** Доказать VP ≠ VNP (алгебраический аналог P ≠ NP)

**Метод:**
- Использовать алгебраическую геометрию
- Изучать замыкания орбит
- Искать препятствия представлений
- Связь с P vs NP через арифметико-булеву конверсию

**Статус:** Долгосрочная программа, активная исследовательская область

**Ссылки:**
- Mulmuley-Sohoni 2001
- Mulmuley "Geometric Complexity Theory" series

### 6.2 Сложность доказательств

**Цель:** Доказать супер-полиномиальные нижние оценки для систем доказательств

**Методы:**
- Изучать резолюцию, системы Фреге
- Использовать комбинаторные принципы
- Связывать с NP vs coNP

**Ключевые результаты:**
- Экспоненциальные нижние оценки для резолюции (Haken)
- Квази-полиномиальные нижние оценки для Frege (не известны)

### 6.3 Дерандомизация

**Цель:** Доказать BPP = P или установить компромиссы сложность-случайность

**Ключевая связь (Kabanets 2001):**
- Если каждый язык в E имеет малые схемы, то P ≠ NP
- Либо дерандомизация, либо разделение

**Методы:**
- Конструировать псевдослучайные генераторы
- Доказывать нижние оценки сложности
- Изучать экстракторы

## 7. Известные барьеры и как их избежать

### 7.1 Барьер релятивизации (BGS 1975)

**Результат:**
- ∃ оракул A: P^A = NP^A
- ∃ оракул B: P^B ≠ NP^B

**Следствие:** Техники, которые релятивизируются, не могут разрешить P vs NP

**Как избежать:**
- Использовать арифметизацию (IP = PSPACE)
- Техники от алгоритмов к нижним оценкам (Williams)
- Использовать неравномерность явно

### 7.2 Барьер естественных доказательств (RR 1997)

**Определение:** "Естественное" доказательство является:
1. Конструктивным (вычислимое свойство)
2. Большим (применимо ко многим функциям)
3. Полезным (против P/poly)

**Барьер:** Если существуют сильные PRG, естественные доказательства не могут доказать нижние оценки

**Как избежать:**
- Использовать неестественные свойства
- Эксплуатировать специфическую структуру
- Алгебраические методы (GCT)

### 7.3 Барьер алгебраизации (AW 2008)

**Расширение:** Обобщает релятивизацию и арифметизацию

**Как избежать:**
- GCT-подобные методы
- Неравномерные разрывы сложности
- Предположения о вычислительной сложности

## Заключение

Этот документ предоставляет обширный инструментарий для подхода к проблеме P vs NP. Ключевые моменты:

1. **Множественные модели вычислений:** Машины Тьюринга, схемы, рандомизированные алгоритмы
2. **Богатые математические инструменты:** Комбинаторика, алгебра, логика
3. **Мощные техники доказательств:** Диагонализация, нижние оценки для схем, коммуникационная сложность
4. **Современные вычислительные инструменты:** SAT-решатели, ассистенты доказательств
5. **Активные исследовательские программы:** GCT, сложность доказательств, дерандомизация
6. **Известные барьеры и стратегии избегания:** Релятивизация, естественные доказательства, алгебраизация

**Ключевое переосмысление для всех инструментов и методологий:**

Поскольку **P ⊆ NP доказано** ([см. доказательства](proofs/p_vs_np_decidable/README.md)), весь вопрос P vs NP сводится к **доказуемости/недоказуемости NP ⊆ P**. Все инструменты и техники должны быть направлены на понимание, может ли каждая задача, верифицируемая за полиномиальное время, также быть решена за полиномиальное время.

Успех требует комбинации глубоких теоретических знаний, вычислительной изобретательности и новых математических идей. Проблема остаётся одним из величайших интеллектуальных вызовов в математике и информатике.
